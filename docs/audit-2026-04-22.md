## 3. Architecture

**Pattern: SOLID-Compliant Service-Oriented + React Context**

```
Components (UI)
    │
    ├── Custom Hooks (useProjects, useTerminal)
    │       │
    │       └── Services (CommandProcessor, ProjectRepository)
    │               │
    │               └── Contracts / Domain Types
    │
    └── ErrorBoundary (class component, lifecycle-safe)
```

**What works well:**

- `ICommandProcessor` interface enables dependency injection in tests
- `ProjectProvider` / `useProjects` pattern cleanly separates data from display
- `animations.ts` as a variant factory eliminates scattered Framer Motion configs
- `siteConfig` and `uiConfig` centralize magic strings and tokens

**What to address:**

- `page.tsx` marks the entire route `'use client'` — consider splitting into a server shell with client islands
- No React `Suspense` boundary around Three.js (if WebGL init hangs, nothing is shown)
- `ErrorBoundary` only catches render errors; async errors in hooks go unhandled

## 4. Performance

### Score: 6.5 / 10

**Implemented optimizations:**

- React Compiler (`reactCompiler: true`) for automatic memoization
- `antialias: false` and `multisampling={0}` on the WebGL canvas
- `dpr={[1, 2]}` dynamic pixel ratio
- Post-processing at reduced height (300px)
- `useMemo` for shader geometry in `TopologyMesh`
- `useRef` for DOM access to avoid re-renders

**Performance gaps:**

| Issue                                                                                  | Impact                         | Fix                                                                                |
| -------------------------------------------------------------------------------------- | ------------------------------ | ---------------------------------------------------------------------------------- |
| `InteractiveTopology` not lazy-loaded                                                  | Blocks initial render          | `next/dynamic` with SSR off + Suspense                                             |
| `getComputedStyle()` called in render loop                                             | Medium GC pressure             | Read once on mount, memoize                                                        |
| `PAdicBackground`: 60 animated particles + rotating 2000×2000px SVG + 3 gradient blobs | High CPU on low-end devices    | Reduce count based on `matchMedia('(prefers-reduced-motion)')` and connection type |
| No loading state while WebGL initializes                                               | User sees blank space for 2–3s | Add skeleton/spinner fallback                                                      |
| No code splitting beyond route level                                                   | All components loaded eagerly  | Dynamic imports for heavy components                                               |
| No `will-change` hints on CSS animations                                               | Potential compositing misses   | Add `will-change: transform` to animated blobs                                     |
| No bundle analysis in CI                                                               | Bundle bloat goes undetected   | Add `@next/bundle-analyzer`                                                        |

**Estimated FCP target:** < 1.5s (currently ~2–3s)

## 5. Accessibility

### Score: 5 / 10

**Good practices:**

- `role="log"` and `aria-live="polite"` on terminal output
- `aria-label` on terminal input
- Semantic heading hierarchy (`h1` → `h2`)
- Keyboard support on terminal (Enter key)
- JetBrains Mono with antialiased rendering

**Gaps — ordered by severity:**

### Critical

- **`prefers-reduced-motion` not respected** — All Framer Motion animations and Three.js loops run regardless. Users with vestibular disorders are directly impacted.
  ```typescript
  // In animations.ts — wrap all variants with:
  const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches
  // Or use the CSS media query in globals.css:
  // @media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; } }
  ```

### High

- **Three.js canvas has no accessible description** — Screen readers encounter a silent `<canvas>`.
  ```tsx
  <Canvas
    role="img"
    aria-label="Interactive 3D topology visualization representing algorithmic data structures"
  />
  ```
- **Interactive 3D mesh has no keyboard control** — Mouse-only interaction excludes keyboard users entirely.

### Medium

- **No WebGL fallback UI** — If Three.js fails to initialize, the canvas section is blank with no message.
- **No skip-navigation link** — Keyboard users must tab through all interactive elements before reaching content.
- **Focus management** — Terminal's `autoFocus` may steal focus from the page unexpectedly on load.

### Low

- Link colors may not meet WCAG AA contrast ratio in all states
- Hero gradient text (`bg-gradient-to-r`) may reduce legibility at small viewport sizes
- Touch targets may be under 44×44px minimum

## 9. WebGL & Animation

### WebGL Score: 8 / 10 | Animation Score: 7 / 10

### WebGL (TopologyMesh + TopologyShaders)

**Strengths:**

- Efficient Simplex noise — computed once per vertex per frame
- Screen-space hover detection via radial `smoothstep` — avoids raycasting overhead
- `AdditiveBlending` for point sprite glow — physically correct
- Uniforms properly expose reactive data (`time`, `mouse`, `color`, `hoverColor`)
- `useMemo` for geometry to avoid re-allocation

**Issues:**

| Issue                                                                    | Severity |
| ------------------------------------------------------------------------ | -------- |
| Fragment discard at `r > 0.5` is hard cutoff — no anti-aliasing          | Low      |
| No LOD system — geometry detail is ternary (`8` vs `16`), not continuous | Low      |
| Magic numbers in shaders should be named constants or uniforms           | Low      |
| No inline GLSL comments explaining the mathematical intent               | Low      |
| CSS variable parse happens in `useEffect` without caching guard          | Medium   |

### Animation (Framer Motion + PAdicBackground)

**Strengths:**

- Centralized `animations.ts` with factory functions — no scattered configs
- `whileInView` for scroll-driven entry — lazy and performant
- Consistent easing curves used throughout

**Issues:**

| Issue                                                                                          | Severity |
| ---------------------------------------------------------------------------------------------- | -------- |
| No `prefers-reduced-motion` check — all animations run unconditionally                         | High     |
| `PAdicBackground`: 60 particles + rotating 2000px SVG + 3 gradient blobs active simultaneously | High     |
| No way to pause/stop infinite loops (battery drain on mobile)                                  | Medium   |
| No animation performance budget or frame-rate monitoring                                       | Low      |

## 10. Mobile & Responsiveness

### Score: 6 / 10

**Responsive breakpoints:**

| Breakpoint     | Usage                                          |
| -------------- | ---------------------------------------------- |
| `md:` (768px)  | ProjectGallery 1→2 cols, header flex direction |
| `lg:` (1024px) | 12-col grid (8+4), sidebar visible             |
| `sm:`          | Not used — all mobile defaults                 |
| `2xl:`         | Not used — no ultra-wide optimization          |

**Layout is fundamentally sound** — single column on mobile, sidebar collapses correctly.

**Mobile-specific concerns:**

| Concern                                                           | Severity |
| ----------------------------------------------------------------- | -------- |
| Three.js shader complexity may cause jank on mobile GPUs          | High     |
| `TopologyMesh` responds to mouse events only — touch is unhandled | High     |
| PAdicBackground infinite animations drain battery                 | Medium   |
| Terminal input may be obscured by the on-screen keyboard          | Medium   |
| No mobile-specific reduced-particle mode                          | Medium   |
| No `sm:` breakpoint — small phones may hit tight layouts          | Low      |
| No `2xl:` — ultra-wide monitors show scaled-up content            | Low      |
| Font (JetBrains Mono via Google Fonts) adds network RTT           | Low      |

**Recommended mobile guard:**

```typescript
const isMobile = window.matchMedia('(max-width: 768px)').matches
const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches
const particleCount = isMobile || prefersReduced ? 15 : 60
```

## 12. Priority Improvements

### P0 — Immediate (1–2 days)

1. **`prefers-reduced-motion`** — Wrap all Framer Motion variants and Three.js animation loops. This affects real users and is a WCAG 2.1 AA requirement.

2. **Lazy-load Three.js** — Wrap `InteractiveTopology` in `next/dynamic({ ssr: false })` with a `Suspense` fallback. Cuts initial bundle and unblocks FCP.

3. **Canvas ARIA label** — Add `role="img"` and `aria-label` to the `<Canvas>` element describing the visualization.

### P1 — Short Term (1 week)

4. **SEO: JSON-LD + robots.txt + sitemap** — Three small files that meaningfully improve discoverability.

5. **OpenGraph image** — Static 1200×630 image for social sharing previews.

6. **Loading state for WebGL** — Show a skeleton or spinner while Three.js initializes.

7. **PAdicBackground performance** — Reduce particle count based on `prefers-reduced-motion` and viewport width. Consider dropping the rotating 2000px SVG on mobile.

8. **CSS variable memoization in TopologyMesh** — Read `getComputedStyle()` once on mount, not on every re-render.

### P2 — Medium Term (2–4 weeks)

9. **Increase test coverage to 60%+** — Prioritize `useTerminal` hook, `ErrorBoundary`, and the `ProjectRepository` service.

10. **Add Playwright E2E tests** — Test the terminal command flow end-to-end; this is the most interactive feature and currently has zero E2E coverage.

11. **CSP headers** — Add `Content-Security-Policy`, `X-Frame-Options`, and `X-Content-Type-Options` in `next.config.ts`.

12. **Bundle analyzer** — Add `@next/bundle-analyzer` and add a CI step to track bundle size over time.

13. **Skip-navigation link** — Add a visually hidden "Skip to content" link as the first focusable element.

### P3 — Nice to Have

14. **Touch support in TopologyMesh** — Map `onPointerMove` touch events to the mouse uniform.
15. **`2xl:` breakpoint** — Constrain max content width on ultra-wide monitors.
16. **Shader documentation** — Add GLSL comments explaining the noise function and glow math.
17. **Font self-hosting** — Move JetBrains Mono to `public/fonts/` to eliminate Google Fonts RTT.
18. **RSC shell** — Convert static sections of `page.tsx` to Server Components; only client-interactive sections need `'use client'`.

## Why OKLCH?

OKLCH is a color space designed for **perceptual uniformity** — a 10% change in any channel produces a roughly equal perceptual difference. This makes it ideal for generating accessible color palettes.

### The problem with hex and RGB

Hex and RGB are device-oriented, not human-oriented. Two colors with the same lightness value in hex can have dramatically different perceived brightness. This makes contrast prediction unreliable.

OKLCH fixes this by separating lightness (L), chroma (C), and hue (H) into independent channels.

## Implementation

### CSS variable tokens

```css
:root {
  --bg-primary: oklch(0.15 0.01 260);
  --bg-card: oklch(0.18 0.015 260);
  --text-primary: oklch(0.95 0.01 260);
  --text-secondary: oklch(0.7 0.02 260);
}
```

### Light theme overrides

```css
.light {
  --bg-primary: oklch(0.99 0.005 85);
  --bg-card: oklch(0.97 0.01 85);
  --text-primary: oklch(0.15 0.02 85);
}
```

## Key takeaways

1. **Set text contrast first**, then build the palette around it
2. **Test every gray pair** — grays are where contrast failures hide
3. **Different themes need different muted values** — don't share them
4. **OKLCH makes palette generation predictable** — no more guessing
